Cub3D – To-Do List (Player → Rendering)

PLAYER (done)

* Get the player’s position from the map
* Store player direction as a vector (based on N/S/E/W)
* Initialize camera plane (for field of view)
* Draw player as a point/small square on the 2D map

2D DEBUG VIEW (VERY IMPORTANT) (done)

* Draw the map as a grid (top-down view)
* Draw walls (green)
* Draw empty space (different color)
* Draw player direction (a line showing where the player looks)

RAYCASTING BASICS

* Loop through each screen column (x)
* Calculate cameraX for each column
* Calculate ray direction (dir + plane * cameraX)

DDA (CORE)

* Get current map square (mapX, mapY)
* Calculate delta distances
* Calculate step (stepX, stepY)
* Initialize side distances
* Perform DDA loop (step through grid)
* Stop when a wall is hit
* Detect which side was hit (X or Y)

DISTANCE & PROJECTION

* Calculate perpendicular wall distance (avoid fish-eye)
* Calculate line height (wall height on screen)
* Calculate draw start and draw end (center vertically)

RENDERING (3D VIEW)

* Draw vertical stripes (one per column)
* Use different colors for different wall sides (for depth)
* Clear image each frame before drawing

PLAYER MOVEMENT

* Move forward/backward
* Rotate left/right
* Update direction vector
* Update camera plane accordingly

COLLISION

* Prevent player from walking through walls
* Check next position before moving

LOOP / FRAME

* Create a game loop (render every frame)
* Redraw everything each frame (no leftovers)

DEBUG (HIGHLY RECOMMENDED)

* Visualize rays on the 2D map
* Print values if something breaks (mapX, mapY, hit, etc.)

OPTIONAL (LATER)

* Texture mapping
